Format items

To format the display of items in the select component, use "formatBy" input.
To format a value, you need a formatter.
There is a set of built-in formatters available, or you can create your own custom formatters.

Some built-in formatters:
  • Field formatter
  • Locale formatter
  • Number formatter (soon)
  • Currency formatter (soon)
To read more about formatters, check the formatters page.
Field formatter:

A field formatter uses a field name of the object to display the value.

[source]="personList1"; formatBy="name" Value: {{ examples.format.value5 | json }}
Locale formatter:

A locale formatter, formats the value according to the current locale.
To use a locale formatter, specify the locale formatter name prefixed with an @ (e.g., "@formatterName").

[source]="[true, false]"; formatBy="@OnOffBoolean" Value: {{ examples.format.value1 | json }}
formatBy="@EnableDisableBoolean" Value: {{ examples.format.value2 | json }}
[source]="Sexuality" formatBy="@SexualityEnum" Value: {{ examples.format.value3 | json}}
[source]="personList1" formatBy="@personFormatter1" Value: {{ examples.format.value4 | json}}
Custom formatter:
Using a formatBy function: [formatBy]="functionName"
[source]="personList1" [formatBy]="personFormatterFunc" Value: {{ value17 | json}}
Compare items:

When working with reference types (e.g., object arrays), you need to specify how to compare the source items with the selected value(s).
By default, the select component uses strict equality (===) to compare items. You can change this behavior using the compareBy input.
You can compare items by a specific field or by a custom function.

[source]="personList1"; compareBy="id" Value: {{ examples.compare.value1 | json }}
[source]="personList1"; [compareBy]="personComparerFunc" Value: {{ examples.compare.value2 | json }}
Custom value writer:

When user selects an item, the select component writes back the selected value to the bound variable.
Sometimes you want customize the value that is written back. This is useful when you want to bind the component to a specific property of an object instead of the whole object.
you can write back the selected value(s) by a specific field or by a custom function.

[source]="personList1" (default, without using writeBy) Value: {{ examples.write.value1 | json }}
[source]="personList1"; writeBy="id" Value: {{ examples.write.value2 | json }}
[source]="personList1"; [writeBy]="personIdWriterFunc" Value: {{ examples.write.value3 | json }}
Note: Using a value writer is useful when binding to database foreign keys.

Filter

To filter the options in the select component, set filterable to true and provide a filter predicate.
You can pass the name of the property to filter by using the filterBy input or a custom filter function.
By default, the select component will filter item values using a default filter function.

[source]="stringArray"; filterable="true" Value: {{ examples.filterExample.value1 | json }}
[source]="personList1"; filterable="true"; filterBy="name" Value: {{ examples.filterExample.value2 | json }}
[source]="personList1"; filterable="true"; [filterBy]="filterFunc" Value: {{ examples.filterExample.value3 | json }}
